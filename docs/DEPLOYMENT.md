# RemitFlow Production Deployment Guide 🚀

**Complete guide for deploying RemitFlow as a production-ready service**

---

## 📋 Pre-Deployment Checklist

### Infrastructure Requirements

- [ ] **Domain name** with SSL certificate (e.g., remitflow.io)
- [ ] **VPS/Cloud server** (minimum: 4 CPU, 8GB RAM, 50GB SSD)
- [ ] **Production PostgreSQL** database (managed service recommended)
- [ ] **Production Redis** instance (managed service recommended)
- [ ] **Stellar mainnet account** with funded oracle keypair
- [ ] **Anchor API credentials** (production keys from partners)
- [ ] **Monitoring service** (Sentry, DataDog, or similar)
- [ ] **Backup strategy** configured

### Security Requirements

- [ ] All secrets generated and stored securely
- [ ] HTTPS/TLS certificates configured
- [ ] Firewall rules set (only 80, 443, 22 open)
- [ ] SSH key-based authentication enabled
- [ ] Regular security updates scheduled
- [ ] Rate limiting configured
- [ ] CORS restricted to production domain

## 🚀 Deployment Methods

### Method 1: Docker Compose (Recommended for Start)

**Best for:** Single server deployment, quick setup

#### Step 1: Server Setup

```bash
# SSH into your server
ssh root@your-server-ip

# Update system
apt update && apt upgrade -y

# Install Docker
curl -fsSL https://get.docker.com | sh
usermod -aG docker $USER

# Install Docker Compose
apt install docker-compose-plugin -y

# Verify installation
docker --version
docker compose version
```

#### Step 2: Clone Repository

```bash
# Create app directory
mkdir -p /opt/remitflow
cd /opt/remitflow

# Clone repository
git clone https://github.com/your-org/remitflow.git .
git checkout main  # or your production branch
```

#### Step 3: Configure Environment

```bash
# Copy production environment template
cp docker/.env.example docker/.env

# Generate secure secrets
./scripts/setup-env.sh

# Edit with production values
nano docker/.env
```

**Production .env Example:**
```env
# Server
NODE_ENV=production

# Database (use managed service URL)
DATABASE_URL=postgresql://user:password@db-host:5432/remitflow

# Redis (use managed service URL)
REDIS_URL=redis://:password@redis-host:6379

# Stellar Mainnet
STELLAR_NETWORK=mainnet
STELLAR_RPC_URL=https://soroban-mainnet.stellar.org
STELLAR_HORIZON_URL=https://horizon.stellar.org
STELLAR_NETWORK_PASSPHRASE=Public Global Stellar Network ; September 2015
SOROBAN_CONTRACT_ADDRESS=YOUR_MAINNET_CONTRACT_ADDRESS

# Secrets (auto-generated by setup-env.sh)
JWT_SECRET=<auto-generated>
SEP10_SERVER_SECRET=<auto-generated>
ORACLE_SECRET_KEY=<auto-generated>

# Domain
CORS_ORIGINS=https://remitflow.io
SEP10_HOME_DOMAIN=remitflow.io
SEP10_WEB_AUTH_DOMAIN=remitflow.io

# Anchor APIs (production credentials)
ANCHOR_CONFIGS=vibrant_colombia|https://api.vibrantapp.com|prod-token-xxx
```

#### Step 4: Build and Deploy

```bash
cd docker

# Create production compose file
cat > docker-compose.prod.yml << 'EOF'
version: '3.8'

services:
  backend:
    build:
      context: ../backend
      dockerfile: Dockerfile
      target: production
    env_file:
      - .env
    restart: unless-stopped
    expose:
      - "3001"
    healthcheck:
      test: ["CMD", "wget", "--spider", "-q", "http://localhost:3001/health"]
      interval: 30s
      timeout: 10s
      retries: 3
    deploy:
      resources:
        limits:
          cpus: '2'
          memory: 2G

  frontend:
    build:
      context: ../frontend
      dockerfile: Dockerfile
      target: production
    env_file:
      - .env
    restart: unless-stopped
    expose:
      - "3000"
    depends_on:
      - backend
    deploy:
      resources:
        limits:
          cpus: '2'
          memory: 2G

  oracle:
    build:
      context: ../oracle
      dockerfile: Dockerfile
      target: production
    env_file:
      - .env
    restart: unless-stopped
    depends_on:
      - backend
    deploy:
      resources:
        limits:
          cpus: '1'
          memory: 1G

  nginx:
    image: nginx:alpine
    ports:
      - "80:80"
      - "443:443"
    volumes:
      - ./nginx.conf:/etc/nginx/nginx.conf
      - ./ssl:/etc/nginx/ssl
    depends_on:
      - frontend
      - backend
    restart: unless-stopped

networks:
  default:
    driver: bridge
EOF

# Build production images
docker compose -f docker-compose.prod.yml build

# Run database migrations
docker compose -f docker-compose.prod.yml run --rm backend pnpm migrate

# Start services
docker compose -f docker-compose.prod.yml up -d

# Check status
docker compose -f docker-compose.prod.yml ps
docker compose -f docker-compose.prod.yml logs -f
```

#### Step 5: Configure Nginx

```bash
# Create Nginx configuration
cat > docker/nginx.conf << 'EOF'
worker_processes auto;

events {
    worker_connections 1024;
}

http {
    upstream frontend {
        server frontend:3000;
    }

    upstream backend {
        server backend:3001;
    }

    # Redirect HTTP to HTTPS
    server {
        listen 80;
        server_name remitflow.io www.remitflow.io;
        return 301 https://$host$request_uri;
    }

    # HTTPS configuration
    server {
        listen 443 ssl http2;
        server_name remitflow.io www.remitflow.io;

        ssl_certificate /etc/nginx/ssl/fullchain.pem;
        ssl_certificate_key /etc/nginx/ssl/privkey.pem;
        ssl_protocols TLSv1.2 TLSv1.3;
        ssl_ciphers HIGH:!aNULL:!MD5;

        # Frontend
        location / {
            proxy_pass http://frontend;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
            proxy_set_header X-Forwarded-Proto $scheme;
        }

        # Backend API
        location /api/ {
            proxy_pass http://backend;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
            proxy_set_header X-Forwarded-Proto $scheme;
        }

        # Health check
        location /health {
            proxy_pass http://backend/health;
        }
    }
}
EOF

# Create SSL directory
mkdir -p docker/ssl

# Copy SSL certificates (from Let's Encrypt or your provider)
cp /path/to/fullchain.pem docker/ssl/
cp /path/to/privkey.pem docker/ssl/

# Restart Nginx
docker compose -f docker-compose.prod.yml restart nginx
```

#### Step 6: Verify Deployment

```bash
# Test frontend
curl -I https://remitflow.io

# Test backend API
curl https://api.remitflow.io/health

# Check all services
docker compose -f docker-compose.prod.yml ps

# View logs
docker compose -f docker-compose.prod.yml logs -f backend
docker compose -f docker-compose.prod.yml logs -f oracle
```

### Method 2: Kubernetes (For Scale)

**Best for:** High availability, auto-scaling, large deployments

#### Step 1: Create Kubernetes Manifests

```yaml
# k8s/deployment.yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: remitflow-backend
  namespace: production
spec:
  replicas: 3
  selector:
    matchLabels:
      app: remitflow-backend
  template:
    metadata:
      labels:
        app: remitflow-backend
    spec:
      containers:
      - name: backend
        image: remitflow/backend:latest
        ports:
        - containerPort: 3001
        envFrom:
        - secretRef:
            name: remitflow-secrets
        resources:
          requests:
            memory: "1Gi"
            cpu: "1000m"
          limits:
            memory: "2Gi"
            cpu: "2000m"
        livenessProbe:
          httpGet:
            path: /health
            port: 3001
          initialDelaySeconds: 30
          periodSeconds: 10
        readinessProbe:
          httpGet:
            path: /health
            port: 3001
          initialDelaySeconds: 5
          periodSeconds: 5
---
apiVersion: v1
kind: Service
metadata:
  name: remitflow-backend
  namespace: production
spec:
  selector:
    app: remitflow-backend
  ports:
  - port: 3001
    targetPort: 3001
  type: ClusterIP
```

#### Step 2: Deploy to Kubernetes

```bash
# Create namespace
kubectl create namespace production

# Create secrets
kubectl create secret generic remitflow-secrets \
  --from-env-file=docker/.env \
  --namespace=production

# Apply manifests
kubectl apply -f k8s/

# Verify
kubectl get pods -n production
kubectl get services -n production
```

## 🔧 Post-Deployment Setup

### 1. Register Mainnet Contract

```bash
# Deploy smart contract to mainnet
cd smart-contracts
cargo build --release --target wasm32v1-none

stellar contract deploy \
  --wasm target/wasm32v1-none/release/remiflow.wasm \
  --source-account remitflow-mainnet \
  --network mainnet \
  --alias remitflow

# Note the contract address
# Update SOROBAN_CONTRACT_ADDRESS in .env
```

### 2. Configure Anchor Partnerships

```bash
# Add production anchors via API
curl -X POST https://api.remitflow.io/anchors \
  -H "Authorization: Bearer ADMIN_TOKEN" \
  -d '{
    "id": "vibrant_colombia",
    "name": "Vibrant Colombia",
    "baseUrl": "https://api.vibrantapp.com",
    "authToken": "production-token",
    "isActive": true
  }'

# Verify anchor is active
curl https://api.remitflow.io/anchors/vibrant_colombia/status
```

### 3. Set Up Monitoring

#### Option A: Basic Health Checks

```bash
# Create monitoring script
cat > scripts/monitor.sh << 'EOF'
#!/bin/bash

# Check services
BACKEND_STATUS=$(curl -s -o /dev/null -w "%{http_code}" https://api.remitflow.io/health)
FRONTEND_STATUS=$(curl -s -o /dev/null -w "%{http_code}" https://remitflow.io)

if [ "$BACKEND_STATUS" != "200" ]; then
  echo "❌ Backend is down!" | mail -s "RemitFlow Alert" admin@remitflow.io
fi

if [ "$FRONTEND_STATUS" != "200" ]; then
  echo "❌ Frontend is down!" | mail -s "RemitFlow Alert" admin@remitflow.io
fi

echo "✅ All services healthy"
EOF

chmod +x scripts/monitor.sh

# Add to crontab (run every 5 minutes)
crontab -e
*/5 * * * * /opt/remitflow/scripts/monitor.sh
```

#### Option B: Sentry Integration

```bash
# Install Sentry SDK
cd backend
pnpm add @sentry/node

# Configure in backend/src/index.ts
import * as Sentry from "@sentry/node";

Sentry.init({
  dsn: process.env.SENTRY_DSN,
  environment: "production",
  tracesSampleRate: 1.0,
});
```

### 4. Set Up Backups

```bash
# Create backup script
cat > scripts/backup.sh << 'EOF'
#!/bin/bash

BACKUP_DIR="/backups/remitflow"
TIMESTAMP=$(date +%Y%m%d_%H%M%S)

# Create backup directory
mkdir -p $BACKUP_DIR

# Backup database
docker exec remitflow-db-1 pg_dump -U postgres remitflow > \
  $BACKUP_DIR/db_$TIMESTAMP.sql

# Backup .env file
cp docker/.env $BACKUP_DIR/env_$TIMESTAMP

# Compress
cd $BACKUP_DIR
tar -czf remitflow_$TIMESTAMP.tar.gz db_$TIMESTAMP.sql env_$TIMESTAMP
rm db_$TIMESTAMP.sql env_$TIMESTAMP

# Keep only last 30 days
find $BACKUP_DIR -name "*.tar.gz" -mtime +30 -delete

echo "✅ Backup completed: $BACKUP_DIR/remitflow_$TIMESTAMP.tar.gz"
EOF

chmod +x scripts/backup.sh

# Add to crontab (daily at 2 AM)
crontab -e
0 2 * * * /opt/remitflow/scripts/backup.sh
```

### 5. Configure Logging

```bash
# Create log rotation
cat > /etc/logrotate.d/remitflow << 'EOF'
/var/log/remitflow/*.log {
    daily
    missingok
    rotate 30
    compress
    delaycompress
    notifempty
    create 0640 root root
    sharedscripts
    postrotate
        docker compose -f /opt/remitflow/docker/docker-compose.prod.yml restart
    endscript
}
EOF
```

## 🔄 CI/CD Pipeline

### GitHub Actions Example

```yaml
# .github/workflows/deploy.yml
name: Deploy to Production

on:
  push:
    branches: [main]

jobs:
  deploy:
    runs-on: ubuntu-latest
    
    steps:
    - uses: actions/checkout@v3
    
    - name: Deploy to server
      uses: appleboy/ssh-action@master
      with:
        host: ${{ secrets.SERVER_HOST }}
        username: ${{ secrets.SERVER_USER }}
        key: ${{ secrets.SSH_PRIVATE_KEY }}
        script: |
          cd /opt/remitflow
          git pull origin main
          docker compose -f docker-compose.prod.yml build
          docker compose -f docker-compose.prod.yml run --rm backend pnpm migrate
          docker compose -f docker-compose.prod.yml up -d
          docker system prune -af
```

## 🛡️ Security Hardening

### 1. Firewall Configuration

```bash
# UFW (Uncomplicated Firewall)
ufw default deny incoming
ufw default allow outgoing
ufw allow 22/tcp    # SSH
ufw allow 80/tcp    # HTTP
ufw allow 443/tcp   # HTTPS
ufw enable
```

### 2. Fail2Ban

```bash
# Install fail2ban
apt install fail2ban -y

# Configure
cat > /etc/fail2ban/jail.local << 'EOF'
[sshd]
enabled = true
bantime = 3600
findtime = 600
maxretry = 3

[nginx-http-auth]
enabled = true
EOF

systemctl restart fail2ban
```

### 3. Regular Updates

```bash
# Create update script
cat > scripts/update.sh << 'EOF'
#!/bin/bash

cd /opt/remitflow

# Pull latest code
git pull origin main

# Rebuild and restart
docker compose -f docker-compose.prod.yml build
docker compose -f docker-compose.prod.yml up -d

# Clean up
docker system prune -af

echo "✅ Update completed"
EOF

chmod +x scripts/update.sh
```

## 🆘 Troubleshooting

### Common Issues

**1. Services won't start**
```bash
# Check logs
docker compose -f docker-compose.prod.yml logs backend

# Verify environment
docker compose -f docker-compose.prod.yml config

# Check disk space
df -h
```

**2. Database connection fails**
```bash
# Test connection
docker compose exec backend pnpm db:test

# Check database logs
docker compose logs db
```

**3. Oracle not fetching rates**
```bash
# Check oracle logs
docker compose logs oracle

# Verify anchor credentials
curl https://api.remitflow.io/anchors/status

# Manually trigger fetch
curl -X POST https://api.remitflow.io/oracle/fetch
```
